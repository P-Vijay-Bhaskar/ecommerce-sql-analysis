# E-commerce data analysis case study using SQL (CTEs, subqueries, GROUP BY, CASE)

This project analyzes e-commerce transactional data to answer real-world business questions using SQL.

# Objectives
- Understand customer purchasing behavior
- Identify high-value and repeat customers
- Analyze sales performance
- Solve business problems using SQL queries

# Tools & Technologies
- SQL
- PostgreSQL / MySQL (any relational DB)
- SQL Concepts Used:
  - CTEs
  - Subqueries
  - GROUP BY
  - CASE statements
  - Joins

# Dataset
The dataset contains:
- Customer details
- Order information
- Product categories
- Sales and revenue data

# Business Questions Solved
- Which customers are repeat buyers?
- Which product categories generate the highest revenue?
- Monthly and category-wise sales trends
- Customer segmentation based on spending

# Key Insights
- Identified high-value customers using aggregations
- Analyzed sales trends using GROUP BY
- Used CASE statements for customer segmentation
- Modular queries written using CTEs for clarity

# How to Use
1. Load dataset into SQL database
2. Run queries from `sql_queries/` folder
3. Review insights in `insights/findings.md`
